Intro by our VP of Engineering Mario KranjčecWatch 'General SW development' | Microsoft Stream

Software development is a complex task and typically follows an SDLC process which can be different depending on the type of project, customer, etc. Software development is typically performed by large number of people (possibly divided into several teams) that work on different components or parts of software. It is important for all team members:

•to understand their role and responsibilities inside a team,

•to understand & follow the project defined processes and

•to be familiar and know how to use the tools that are being used in the team (e.g. Git, JIRA).


The following are essential SDLC stages:

•Problem Definition

•Plan

•Design

•Build

•Test

•Deploy

•Maintain


There are several factors that one needs to consider when working on large software solutions or products including:

•Architecture and design constraints

•Operability (system availability, stability, reliability, recoverability, audit trail, capacity, data retention)

•Performance (operational performance, scalability, load balancing, efficiency)

•Usability

•Security

•Internationalization and Localization (I18N/L10N)

•Supportability and maintainability


In enterprise world product maintenance is a serious job which typically involves a group of people which are not software developers. It is therefore very important to produce software which can be supported and maintained by someone else. This involves thinking about providing enough audit trails, logs, metrics, dashboards, etc. already when designing and implementing the software.

Software engineers are responsible to write quality software and must therefore think about software testabilityearly in the SLDC. One of the main tasks is to automate things as much as possible, including code quality checks and software testing. Software testing can be categorized into Functional, Non-Functional and Regression testing. One of the core developer tasks is to make sure to implement Unit and Integrations tests together with new functionality during development phase. Automating unit and integration testing can significantly improve software quality and minimize manual testing efforts. It goes also hand in hand with the CI/CD principles that were already mentioned in this document.

Content

- What is Software IP?

• Software intellectual property, also known as software IP, is a computer code or program that is protected by law against copying, theft, or other use that is not permitted by the owner. Software IP belongs to the company that either created or purchased the rights to that code or software. Any unauthorized use of it by someone else is illegal.

•


-What is Non-Disclosure Agreement?

NDA protects the owner of the IP (typically our customers) from the other party (typically us) developing the same IP and competing against the IP owner on market with it.


-Use of 3rd party software:

• 3rd party software usage must be documented and approved by legal Open source licenses must be approved by legal team and any obligations must be satisfied (e.g. distributing open source component license with the product)
